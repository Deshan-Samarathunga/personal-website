Reflective Journal

Part 1: Professional Identity & Communication

Building a Professional Identity

PPW helped me understand that professional identity is built through the way I present my skills, attitude, learning progress, and project evidence. As an IT undergraduate, I need to show more than academic work. My GitHub profile, LinkedIn presence, portfolio, and documentation should communicate that I am serious about becoming a backend-focused developer. This lesson is important because employers often judge readiness through consistency, professionalism, and the ability to explain real work clearly.

The Importance of Written Communication

I learned that professional writing must be clear, respectful, accurate, and well structured. Emails, reports, CVs, and application messages create an impression before any interview or meeting happens. In my future backend-focused IT career, written communication will also be important for documenting APIs, explaining database decisions, writing setup instructions, and sharing progress with teams. I want to improve this skill by writing shorter, clearer, and more organized technical explanations.

Presentation and Communication Skills

Presentation activities showed me that communication is not only about speaking, but also about organizing ideas for the audience. Backend work can become technical very quickly, so I need to explain APIs, data flows, system decisions, and project outcomes in a simple way. PPW encouraged me to prepare before speaking, use examples, and build confidence. I will apply this when presenting projects, discussing technical decisions, and explaining my learning journey during interviews.

Reflective Journal

Part 2: Interview Preparation & Technical Growth

CV, LinkedIn, GitHub, and Career Networking

PPW helped me understand that a CV, LinkedIn profile, and GitHub account work together as professional evidence. For an IT student, GitHub is especially useful because it can show project structure, code quality, README files, and backend learning progress. I plan to keep improving my repositories, describe my projects clearly, and use LinkedIn to connect with people in the industry.

Interview Preparation and Employer Expectations

I learned that employers look at technical ability, problem-solving, communication, teamwork, attitude, and cultural fit. Interview preparation should include company research, reviewing technical concepts, preparing examples, and avoiding vague answers. This is useful for my backend-focused career because I need to explain how I solve problems, design APIs, handle data, and learn from challenges.

Non-Verbal Communication and STAR Method

PPW also highlighted the importance of eye contact, posture, hand gestures, voice modulation, and virtual interview setup. The STAR method gave me a practical structure for answering behavioral questions using Situation, Task, Action, and Result. I can use this method to explain project work, debugging experiences, teamwork, and how I handled technical problems.

What Comes Next

Overall, PPW helped me understand professional identity, interview preparation, communication, documentation, and career readiness. My next step is to continue improving my backend knowledge, build stronger projects, document my work properly, and prepare for internship opportunities with more confidence and discipline.
